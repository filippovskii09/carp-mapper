# 10. GPS Precision

GPS у браузері не гарантує сантиметрову точність. На iPhone або Android точність залежить від:

- GPS/GLONASS/Galileo fix;
- Wi-Fi positioning;
- мобільних веж;
- дозволів браузера;
- видимості неба;
- енергозбереження;
- того, чи app відкритий як PWA або у браузері.

## Що змінили

Раніше app брав перший результат:

```ts
navigator.geolocation.getCurrentPosition(...)
```

Це часто погано, бо перший fix може бути Wi-Fi/IP приблизним.

Тепер app використовує `watchPosition` до 12 секунд і вибирає найточніший результат:

```ts
if (!bestFix || fix.accuracy < bestFix.accuracy) {
  bestFix = fix;
}
```

Якщо точність досягла 12 м або краще, app завершує пошук раніше.

## Accuracy circle

Store зберігає:

```ts
anchorAccuracy: number | null;
```

Mapbox малює коло похибки навколо anchor. Це важливо: користувач бачить не тільки точку, а й довіру до неї.

Якщо коло велике, app чесно підказує:

```txt
GPS неточний. Використай “Вказати на мапі”.
```

## Чому ручний режим все одно потрібен

Якщо пристрій віддає точність ±80 м, app не може математично зробити її ±2 м. У такому випадку найкращий UX — не приховувати похибку, а дозволити користувачу поставити точку руками по супутниковому орієнтиру.

## Практична рекомендація

Для найкращої точності:

- відкрий app на телефоні, не на desktop;
- дозволь precise location;
- вийди на відкрите місце;
- зачекай 5-12 секунд після натискання `Я тут (GPS)`;
- якщо коло похибки велике, натисни `Вказати на мапі`.
