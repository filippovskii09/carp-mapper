# 09. Field-first Marker Logging

Цей refactor адаптує CarpMapper під реальний процес маркування в короповій ловлі.

## Що було не так

Початкова форма питала:

```txt
Дистанція, м
Азимут
Глибина
Структура
```

Це технічно правильно, але не повністю відповідає польовому процесу. Рибалка часто мислить так:

```txt
22 витки + 1 м
орієнтир: роздвоєне дерево
глибина: 4.2 м
дно: мул із переходом на гравій
```

Тому UX має приймати дані у мові рибалки, а не змушувати його вручну конвертувати все у метри.

## Дистанція у витках

Додано режим дистанції:

```txt
Метри | Витки
```

У режимі `Витки` користувач вводить:

- кількість витків;
- залишок у метрах;
- дистанцію між кілочками.

Сервіс:

```txt
src/services/DistanceService.ts
```

рахує:

```ts
distance = wraps * pegDistanceMeters + remainderMeters;
```

Чому це важливо: рибалка може повторити точку на робочому або сподовому вудилищі без ментальної арифметики на березі.

## Компенсація глибини

При закиданні робочої оснастки вантаж після кліпси тоне по дузі в напрямку берега. Чим більша глибина, тим більша практична потреба додати дистанцію.

MVP використовує практичну рекомендацію:

```ts
compensationMeters = min(depthMeters * 0.3, pegDistanceMeters);
```

Для 6 м глибини і 3.6 м між кілочками:

```txt
6 * 0.3 = 1.8 м = 0.5 витка
```

Це не претендує на ідеальну фізичну модель, але дає корисну польову підказку: сподова і робоча дистанції можуть відрізнятися.

## Орієнтир

Marker тепер має поле:

```ts
horizonMarker: string;
```

Це текстовий лог візуального орієнтира: дерево, труба, будинок, розрив у кронах.

Чому це важливо: дистанція без вектора не дає повторюваної точки. Витки відповідають на питання “наскільки далеко”, орієнтир — “куди саме”.

## Тактильні підказки

Додано:

```txt
src/features/markers/substrateAdvice.ts
```

Для кожної структури дна app показує:

- що має відчуватись у бланк;
- який монтаж розглянути;
- як думати про прикормку.

Це робить app корисним для новачків, але не заважає профі: профі швидко вибирає структуру, новачок отримує контекст.

## Великі кнопки структури

Select замінено на великі кнопки:

```txt
Мул | Пісок | Гравій | Глина | Трава
```

Чому: у польових умовах select незручний. Великі кнопки краще працюють з мокрими руками, на сонці і однією рукою.

## Дані marker

Marker зберігає і абсолютні координати, і польову математику:

```ts
distance: number;
distanceMode: 'meters' | 'wraps';
wrapDistance: WrapDistance | null;
workRodDistance: number;
workRodWraps: number | null;
horizonMarker: string;
```

Чому: координати потрібні мапі, а wraps/horizon потрібні рибалці для повторення точки.

## Головний висновок

CarpMapper рухається від “карта з точками” до “тактичний польовий логер”. Додаток приймає дані так, як їх реально отримують на березі: витки, орієнтир, структура, глибина і практична компенсація.
